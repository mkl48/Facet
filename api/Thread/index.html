Thread

A small pool of Actors all running a clone of the same template, dispatched to round-robin. Build one with Thread.new; nothing else here is meant to be called directly.

Functions​

new​

</>
Thread.new(
template: Script | LocalScript,
parent: Instance,
size: number?,
readyTimeout: number?
) → Thread?

Builds a pool of size Actors (default 4), each running a clone of template parented under parent. Blocks (up to readyTimeout seconds, default 5) until every worker fires its own "Ready" BindableEvent, since a freshly parented Script/LocalScript doesn't start running the instant it's cloned in - it resumes on a later tick, after this call would otherwise have already returned. Returns nil instead of throwing if any worker fails to start in time, so callers can fall back to running work inline rather than crashing.

Map​

</>
Thread:Map(
message: string,
argsList: {{any}}
) → {any}

Dispatches one message per entry in argsList across the pool round-robin and returns the single-value replies in the same order argsList was given (not the order replies actually arrive in). Each worker is expected to SendMessage-style reply by firing the BindableEvent it's handed as the first extra argument with exactly one value.

Destroy​

</>
Thread:Destroy() → ()

Destroys every Actor in the pool. The pool isn't reusable after this.
